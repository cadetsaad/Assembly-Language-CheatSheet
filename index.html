MS-DOS Assembly Language Cheat Sheet

Interrupts

Interrupts are software-generated events that can be triggered by both hardware and software. In MS-DOS Assembly Language, interrupts are commonly used for interacting with the operating system and hardware components.

Function Description Example
Int 21h DOS Function Calls - Allows access to various DOS functions such as file operations, memory allocation, and program termination. INT 21h
Int 10h Video Services - Provides functions for controlling video display, including setting cursor position, changing screen colors, and displaying characters. INT 10h
Int 16h Keyboard Services - Enables interaction with the keyboard, allowing the program to read keypresses and control keyboard LEDs. INT 16h

Data Movement

Data movement instructions are used to transfer data between memory locations and registers.

Instruction Description Example
MOV Move Data - Copies data from one location to another. MOV AX, BX
LEA Load Effective Address - Loads the offset address of the source operand into the destination operand. LEA SI, array

Arithmetic and Logic

Arithmetic and logic instructions perform mathematical and logical operations on data.

Instruction Description Example
ADD, SUB, INC, DEC Addition, Subtraction, Increment, Decrement - Perform arithmetic operations on operands. ADD AX, BX
MUL, DIV Multiplication, Division - Perform multiplication and division operations. MUL CX
AND, OR, XOR, NOT Bitwise Operations - Perform logical operations on operands. AND AX, 0xFF

Control Flow

Control flow instructions alter the sequence of program execution based on conditions or direct program flow to specific locations.

Instruction Description Example
JMP Unconditional Jump - Transfers control to a specified address unconditionally. JMP label
JZ, JNZ, JC, JNC Conditional Jumps based on Flags - Perform conditional jumps based on the state of processor flags. JZ label
JS, JNS Conditional Jumps based on Sign Flag - Jumps based on the state of the sign flag. JS label
CALL, RET Procedure Calls and Returns - CALL is used to call a procedure or subroutine, and RET is used to return from a procedure. CALL subroutine

Procedures (PROC) and Macros (MACRO)

Procedures and macros provide a way to modularize code and reuse it across the program.

In assembly language, a procedure is a named block of code that performs a specific task. It can be called from other parts of the program.

A macro is a named block of code that is expanded inline wherever it is invoked. Macros are often used for code repetition or to define complex instructions.

Here's an example of defining and using a procedure and a macro:

        PROC procedure_name
            ; Procedure body
            ret
        ENDP

        MACRO macro_name parameters
            ; Macro body
        ENDM
